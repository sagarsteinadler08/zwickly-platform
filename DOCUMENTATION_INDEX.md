## ğŸš€ USING THE DOCUMENTATION

### Import to Confluence

**Step 1: Copy Content**
```bash
# Open the file you want to import
cat PRODUCT_DOCUMENTATION_CONFLUENCE.md
# or
cat TECHNICAL_DOCUMENTATION_CONFLUENCE.md
```

**Step 2: Create Confluence Page**
1. Go to your Confluence space
2. Click "Create" â†’ "Blank Page"
3. Give it a title: "Zwickly Platform - Product Documentation"

**Step 3: Paste & Format**
1. Click "Edit" button
2. Switch to "Markup" or "Source" view
3. Paste the content
4. Confluence will auto-render all macros:
   - `{panel}` becomes colored boxes
   - `{code}` becomes syntax-highlighted blocks
   - `{table}` becomes formatted tables
   - `{info}/{warning}/{tip}` become alert boxes
   - `{toc}` becomes clickable table of contents

**Step 4: Add Screenshots**
1. Click where you want to insert image
2. Type `/image` and select "Image"
3. Upload the PNG files
4. Resize as needed

**Step 5: Publish**
1. Click "Publish" button
2. Set permissions (who can view)
3. Share link with team!

### For Non-Confluence Users

If you don't use Confluence, the `.md` files render beautifully on:
- **GitHub** (automatic rendering)
- **GitLab** (automatic rendering)
- **Notion** (import as Markdown)
- **Obsidian** (local wiki)
- **VS Code** (with Markdown Preview)
